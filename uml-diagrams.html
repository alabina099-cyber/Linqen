1. Acteurs du Système

Le système LinkedIn Agent met en relation quatre acteurs principaux pour automatiser la prospection sur LinkedIn tout en respectant les limites de la plateforme.
ActeurTypeDescription
UtilisateurPrincipalLe professionnel qui configure les campagnes, approuve les actions et consulte les résultats de prospection.
Agent IASystèmeAgent conversationnel basé sur OpenAI (LangChain). Analyse les demandes, utilise des outils (tools) pour rechercher, analyser et contacter les prospects.
Extension ChromeSystèmeExtension navigateur qui exécute les actions LinkedIn (recherche, visite de profil, envoi de messages/connexions) en simulant le comportement humain.
LinkedInExternePlateforme sociale professionnelle. Source des données de prospects et canal de communication.

Acteurs : Utilisateur, Agent IA, Extension Chrome, LinkedIn
Précondition : L'utilisateur est connecté à LinkedIn via l'extension Chrome.
Postcondition : Les prospects trouvés sont sauvegardés en BDD et les messages sont en attente d'approbation.

5. Diagramme de Séquence : Workflow de Prospection Autonome

Ce diagramme montre le flux complet d'une prospection autonome : de la demande utilisateur en langage naturel jusqu'à l'envoi des messages, en passant par la recherche LinkedIn, le scraping des résultats, et l'approbation des actions. 
sequenceDiagram
    actor U as Utilisateur
    participant F as Frontend
(Next.js)
    participant AG as Agent IA
(LangChain + OpenAI)
    participant DB as Base de Données
(PostgreSQL / Neon)
    participant EXT as Extension Chrome
(background.js)
    participant CS as Content Script
(content.js)
    participant LI as LinkedIn

    Note over U,LI: Phase 1 — Demande et Recherche
    U->>F: "Cherche les ingénieurs IA dans mon réseau
et envoie un message de proposition"
    F->>AG: chat(message, context)
    AG->>AG: Analyse la demande (LLM)
    AG->>DB: checkRateLimit("search")
    DB-->>AG: {allowed: true}
    AG->>DB: INSERT linkedin_actions_queue
(type=search, url=linkedin.com/search?network=F&keywords=...)
    DB-->>AG: {id: 52, status: pending_approval}
    AG-->>F: "Recherche créée (#52).
Veuillez l'approuver."
    F-->>U: Affiche l'action en attente

    Note over U,LI: Phase 2 — Approbation et Exécution
    U->>F: Clique "Approuver"
    F->>DB: UPDATE status = 'approved' WHERE id=52
    EXT->>DB: pollAndExecuteActions()
GET /api/linkedin-actions?status=approved
    DB-->>EXT: [{id:52, type:search, url:...}]
    EXT->>EXT: executeSearch(action)
    EXT->>LI: chrome.tabs.create(searchUrl)
    LI-->>EXT: Page de résultats chargée
    EXT->>CS: sendMessage(SCRAPE_SEARCH_RESULTS)
    CS->>LI: querySelectorAll (scraping DOM)
    LI-->>CS: Données des profils
    CS-->>EXT: {type: SEARCH_RESULTS, data: [...profiles]}

    Note over U,LI: Phase 3 — Sauvegarde des Prospects
    EXT->>F: POST /api/prospects/bulk
[{name, role, location, linkedin_url}, ...]
    F->>DB: UPSERT prospects (ON CONFLICT linkedin_url)
    DB-->>F: {imported: 8}
    EXT->>DB: UPDATE linkedin_actions_queue
SET status='completed', result={profiles}
    EXT->>EXT: Fermer l'onglet LinkedIn

    Note over U,LI: Phase 4 — Récupération et Envoi en Masse
    U->>F: "Continue" / L'agent reprend
    F->>AG: chat("continue")
    AG->>DB: get_search_results()
SELECT FROM prospects WHERE notes LIKE '%search%'
    DB-->>AG: [{name:"Alice", role:"ML Engineer"}, ...]
    AG->>AG: bulk_send_messages()
Personnalise {name}, {role}, {company}
    AG->>DB: INSERT linkedin_actions_queue × N
(type=send_message, status=pending_approval)
    DB-->>AG: {queued: 8 messages}
    AG-->>F: "8 messages créés. Approuvez-les."
    F-->>U: Affiche les 8 messages en attente

    Note over U,LI: Phase 5 — Approbation et Envoi des Messages
    U->>F: Approuve les messages
    F->>DB: UPDATE status = 'approved'
    EXT->>DB: poll (GET approved)
    loop Pour chaque message
        EXT->>LI: Ouvrir profil prospect
        EXT->>CS: SEND_MESSAGE
        CS->>LI: Clic "Message" → Saisie → Envoi
        LI-->>CS: Message envoyé
        CS-->>EXT: ACTION_COMPLETED
        EXT->>DB: UPDATE status='completed'
        EXT->>EXT: delay(30-90s) anti-ban
    end
    EXT-->>U: Tous les messages envoyés
