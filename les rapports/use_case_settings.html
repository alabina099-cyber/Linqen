Settings — Use Case Specification

Linqen LinkedIn AI Prospecting Agent
Use Case Settings
Actor User
Preconditions User authenticated via valid JWT session.
Postconditions Configuration preferences persisted to user settings.
Main Scenario
[S1] Manage Account

1.1 User navigates to Settings, selects Account tab.

1.2 System displays profile data and LinkedIn connection status.

[S2] Configure AI Model

2.1 User selects Agent tab, opens AI Model section.

2.2 System lists available models with availability status.

2.3 User selects model and message tone (professional, friendly, formal).

2.4 User toggles auto-reply, language auto-detection, and human like behaviour.

2.5 User sets working hours (start/end), timezone, and active working days.

2.6 User configures min/max delay between actions for human-like pacing.

2.7 System validates ranges and saves.

2.8 User toggles Simulate Human Behavior.

2.9 User toggles Randomize Delays to vary inter-action timing.

2.10 System persists flags; Chrome extension reads them during action execution.

[S3] Manage Notifications

3.1 User opens Notifications tab.

3.2 User toggles alert limit, statistic resume, new responce and conversion alert.

[S4] Manage Appearance

4.1 User selects Appearance tab.

4.2 User picks theme (light / dark / auto), language (fr / en), date format (DD/MM/YYYY), and time format (12h / 24h).

4.3 System applies theme.
Alternatives A1 Not authenticated → Redirect to Login page.
S4-4a System theme preference auto-detected via prefers-color-scheme when set to auto.
Exceptions S1-10a DB connection error during save → "server error"; changes not persisted.
S4-10a Unexpected error → Cancel, notify user, log to error_logs.
