Manage Users — Use Case Specification

Qlinqen LinkedIn AI Prospecting Agent
Use Case Manage Users
Actor Admin
Preconditions Admin authenticated via valid LinkedIn cookie.
Postconditions Team member account view, created, modified, activated/inactivate, deleted
Main Scenario
[S1] View Users

1.1 Admin navigates to Users section.

1.2 System displays team members linked to Admin.

1.3 Renders cards with status and actions (Edit, Toggle, Delete).

[S2] Create User

2.1 Admin clicks Create User; form appears (name, email, password, company).

2.2 Admin fills and submits; system validates (XSS, email, password) and checks 10-user limit.

2.3 System hashes password, inserts with role='user' and admin_id.

2.4 Displays confirmation and refreshes list.

[S3] Edit User

3.1 Admin clicks Edit; pre-filled form appears.

3.2 Admin modifies and saves; system validates ownership, updates record.

[S4] Activate/Inactivate

4.1 Admin toggles status switch; system validates ownership, updates is_active.

[S5] Delete User

5.1 Admin clicks Delete and confirms; system validates ownership, executes DELETE.
Alternatives A1 Not authenticated → Redirect to Admin Login.
S2-4a Cancels creation → Discards, returns to list.
S2-5a 11th user attempted → Blocked: "Limit of 10 users reached."
S3-3a Required fields empty → Validation error.
S4-4a Last active user inactivated → Warning, no active members remain.
Exceptions S2-10a DB error → "Erreur serveur"; not saved.
S2-10b Duplicate email → "This email address is already in use."
S2-10c Missing migration → "Missing DB migration."
S3-10a User not owned → 404 "User not found or unauthorized."
S5-10a Unexpected error → Cancel, notify, log to error_logs.
